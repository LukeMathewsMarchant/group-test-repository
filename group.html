What is GitHub?

GitHub is a cloud-based platform that uses Git for version control. It allows developers to store, manage, and track changes to their code, collaborate with others, and contribute to open-source projects.

üí° Pro Tip: Think of GitHub as Google Drive for developers - but with superpowers for tracking every change made to your files!

üìÇ Working with Repositories

What is a Repository?

A repository (or "repo") is like a project folder that contains all your project files and the entire history of changes made to those files.

Creating Your First Repository

üìù Repository Creation Steps

Cloning creates a local copy of a repository on your computer.

git clone https://github.com/username/repository-name.git cd repository-name
üí° Quick Tip: Use HTTPS for public repos, SSH for private repos you own. SSH is faster and more secure!

üå≥ Branching and Merging

Why Use Branches?

Branches allow you to work on different features or experiments without affecting the main codebase. Think of them as parallel universes for your code!

Working with Branches

# Create and switch to a new branch git checkout -b feature-branch # Or using newer syntax git switch -c feature-branch # List all branches git branch # Switch between branches git switch main git switch feature-branch # Push new branch to GitHub git push -u origin feature-branch

Merging Branches

# Switch to main branch git switch main # Merge feature branch git merge feature-branch # Delete branch after merging git branch -d feature-branch git push origin --delete feature-branch
‚ö†Ô∏è Merge Conflicts: When two branches change the same lines of code, Git can't automatically merge them. You'll need to manually resolve conflicts by editing the files and choosing which changes to keep.

Branch Naming Conventions

feature/user-authentication bugfix/login-error hotfix/security-patch docs/api-documentation

ü§ù Collaboration and Pull Requests

Forking a Repository

Forking creates your own copy of someone else's repository under your GitHub account.

üìù Contributing to Open Source

Creating a Pull Request

A Pull Request (PR) is a way to propose changes to a repository. It allows others to review your code before it gets merged.

üí° Great Pull Request Tips:
  • Write a clear title and description
  • Reference related issues
  • Keep changes focused and small
  • Test your changes thoroughly

Code Review Process

1. Submit Pull Request 2. Automated tests run (if configured) 3. Team members review code 4. Address feedback and make changes 5. Approve and merge 6. Delete feature branch

Keeping Your Fork Updated

# Add original repo as upstream git remote add upstream https://github.com/original/repo.git # Fetch latest changes git fetch upstream # Switch to main and merge git switch main git merge upstream/main # Push updates to your fork git push origin main

üöÄ Advanced GitHub Features

GitHub Issues

Issues are used to track bugs, feature requests, and other tasks. They're like a to-do list for your project.

# Link commits to issues git commit -m "Fix login bug - closes #123" # Reference issues in commits git commit -m "Update user validation (see #45)"

GitHub Actions (CI/CD)

Automate testing, building, and deployment of your code with GitHub Actions.

# .github/workflows/ci.yml name: CI on: [push, pull_request] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Run tests run: npm test
